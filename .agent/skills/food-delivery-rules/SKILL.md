# Архітектура проєкту "Доставка їжі" (FastAPI + SQLite)

Ти розробляєш студентську курсову роботу. Суворо дотримуйся наступних правил архітектури:

## 1. Багаторівнева архітектура
Код має бути жорстко розділений на шари: **Controllers → BusinessLogic (Services) → Repositories → Database**.

## 2. База даних
Використовується виключно файлова база **SQLite** та **SQLAlchemy (ORM)**. Усі класи бази даних (Models) мають зберігатися у папці `Domain/Models/`. У нас 5 таблиць: **Users, Restaurants, Dishes, Orders, OrderItems**.

## 3. Заборона прямого доступу
Прямі запити до бази даних (через SQLAlchemy session) у контролерах **суворо ЗАБОРОНЕНІ**. Використовуй виключно шар `Repositories`.

## 4. Контролери (Лабораторна 3)
Файли в папці `Controllers` містять лише FastAPI роутери. Вони **не повинні містити жодної бізнес-логіки**. Їхня єдина задача — прийняти запит і делегувати його в Service.

## 5. Бізнес-логіка (Лабораторна 4)
Уся логіка (розрахунок `total_price` для замовлень, перевірка статусів тощо) лежить у класах Service у папці `BusinessLogic`. Сервіси підключаються до контролерів через **Dependency Injection** (через `Depends()`).

## 6. Валідація (ViewModels)
Для будь-яких вхідних/вихідних даних ендпоінтів обов'язково використовуй **Pydantic схеми**. Вони мають зберігатися у папці `Domain/ViewModels/`.
